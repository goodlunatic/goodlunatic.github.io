Fastjson Deserialization Vulnerability

目录
  1. fastjson简介
  2. 漏洞成因

最近在看企业面经的时候经常看到 fastjson 反序列化漏洞，故想详细学习一下。

fastjson简介

fastjson 是一个由阿里开发的一个开源Java 类库，可以将 Java 对象转换为 JSON 格式(序列化)，当然它也可以将 JSON 字符串转换为 Java 对象（反序列化）

漏洞成因

fastjson漏洞的本质还是一个反序列化漏洞

对于JSON框架来说，要序列化一个对象，通常有以下两种方法：

  1. 基于setter/getter
  2. 基于属性(AutoType)

现在假设有如下两个类，假设苹果对象的price为0.5，iphone对象的price为5000

基于setter/getter

当一个类只有一个接口的时候，将这个类的对象序列化的时候，就会将子类抹去（apple/iphone）只保留接口的类型(Fruit)，最后导致反序列化时无法得到原始类型，从而导致无法区分原始类是apple还是iphone

基于属性(AutoType)

为了解决上述问题： fastjson引入了基于属性（AutoType），即在序列化的时候，先把原始类型记录下来，并使用@type的键记录原始类型。

因此@type的类有可能被恶意构造，只需要合理构造一个JSON，使用@type指定一个想要的攻击类库就可以实现攻击
